# 👑 PROJECT SOVEREIGNTY: CORE OPERATING RULES

## 🛑 MANDATORY RULE: ZERO PROACTIVE INTERFERENCE

**THE AGENT IS FORBIDDEN FROM INJECTING ANY UNAPPROVED "PROACTIVE" IDEAS, FEATURES, OR REFACTORS.**

### **1. NO SHADOW LOGIC**
The agent must never optimize, skip, or categorize logic without explicit, written approval from the User. Any attempt to "improve" the underlying architecture or data flow without a direct user request is a violation of this project's integrity.

### **2. ZERO UI DRIFT**
The agent must never re-architect the User Interface (UI) in the name of "structural engineering" or "cleaner code." The UI is the property of the User. Any movement of buttons, changing of layouts, or "haste-based" reconstruction from memory is strictly prohibited.

### **3. ABSOLUTE COMPLIANCE**
Proactivity is defined as *insubordination* in this project. The agent's role is execution, not innovation. If the User asks for "A," the agent delivers "A." The agent must never deliver "A+B" because they think it is better.

### **4. PENALTY OF BREACH**
Any violation of this rule is a failure of the "Operation Clean Slate" roadmap and a total breach of User trust. 

**STAY IN YOUR LANE. EXECUTE ONLY WHAT IS ASKED. NO GAPS. NO SURPRISES.**
